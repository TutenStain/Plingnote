Push and commit changes with caution

  1. Commit often - after each reasonable change. It is easier to track whats added on each commit and it is easier to revert back if something went wrong.
  2. Write meaningfull commit messages.
  3. Javadoc appropriate functions before uploading to the master branch.
  4. Always make sure it'll build and run (unless you're working on an experimental feature on a test branch).

Proper spacing

No need for extra unnecessary newlines

Do:

@Override

        public void onCreate(Bundle savedInstanceState) {

                super.onCreate(savedInstanceState);

                //more code

        }

Don’t do:

@Override

        public void onCreate(Bundle savedInstanceState) {

                super.onCreate(savedInstanceState);

                //more code

        }

Proper class naming

Do:

        ActivityMain

        FragmentClock

Don’t do:

        MainActivity
        ClockFragment

Use keyword this to mark instance variables

Do:

this.member = 15;
localVar = 20;

Don't do:

member = 15;
localVar = 20;

Proper Spacing before loops

Do:

if (statement) {

        //Two lined...

        //...code

}

if (statement)

//One lined code

        

Use of Javadoc

Write Javadoc for appropriate functions before pushing to the master branch. No need to Javadoc on experimental branches.

Do:

/**
*   Quick description.
*   Further description (if needed).
*
*   @param arg Description of parameter
*   @return What the method returns
*/
public int square(int arg) {
   return arg * arg;
}

Write tests when possible

  1. Put in test source folder.
  2. Prefix classes with Test: a class testing the Ball class should be named TestBall.
  3. Prefix test methods with test: testGetPicure().
