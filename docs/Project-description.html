What

PlingNote is an application where users write, edit and view notes and tag them with a location on a map. Users can add an alarm to each note which is triggered when the user is approaching the position of a note’s location tag.

The user can alternate between two views - map and list. These are different ways of viewing the notes.

Why

The application will make it possible for the user to get notifications at a given location. Let’s say, for example, that you’ve written a shopping list tagged at a nearby store. PlingNote makes sure that you won’t forget the note when you’re approaching it.

How

The application will consist of several activities that has the purpose of displaying a map, a list of notes and a view for creating or editing notes. The user’s notes and their settings will be preserved in a database.

License

Our license will be the GNU General Public License version 3. We find this license appropriate as free software easily can reach out to a large amount of people. In addition, should any other students wish to study or improve our code, they will have that opportunity.

One of the reasons why we chose GPLv3 instead of other open source licenses is its clarity regarding the user’s freedom. The terms of GPLv3 also state very clearly that once software is licensed under its terms, redistributed versions of that software will be licensed under those terms as well.

Another reason to why we are into using GPLv3 is that we think that this application has a great idea behind it and we want to see other people taking it to higher levels. Since this is such a great idea it’s most likely that there will be a price taken for the application and we have the thoughts that this should be a free app that anyone can improve.

Open source licenses can be found in the application settings view.

Milestones

Milestone 1

Basic navigational GUI and a place to write simple notes that will get saved to the database. On program re-launch the user should be able to see the previously entered note.

Milestone 2

The user can navigate through added notes in a simple list view, where the note’s title and a snippet of the content is visible. A long click on a note brings up an action bar, and the is then allowed to select more notes and delete them (or dismiss the action bar).

Milestone 3

Search functionality implemented. The ability to search for notes and view them.

Milestone 4

 Application wide settings where the user can change some settings regarding the application. From here the user will also be able to view information about the application and send feedback. A settings menu from the noteview with the ability to delete and/or clear note settings.

Milestone 5

Saved notes can be viewed on a map, where they are placed on the entered coordinate. If the user wishes, he/she can add a new note at the current position.

Milestone 6

The user can attach useful stuff to a note through a bar on the bottom of the edit view. Add an alarm that is triggered when approaching the entered location, add an image to make it easier to find the note on the map, and tag the note with a category.

Features

Map View

A place where the user can see notes placed on the map based on where they are tagged. From here the user can also choose a place to write a note.

Snotebar

The actionbar on the bottom of the view which is shown when a user is in the edit note view. This bar holds settings related to the note. From here you can set position to trigger the note, set an alarm, attach picture to the note and select a note category. These will be big squared buttons which will feature live preview of the settings. E.g. the map icon will be dynamic and show terrain surrounding the selected position.

List View

See a preview of all notes in a listview. In this view it is also possible to see basic information about the note. Like if the note is tagged with a location or if the note have any attached picture. The user can choose a note to read and also edit.
